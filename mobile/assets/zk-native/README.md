# ZK Circuit Assets — Native Prover

This directory holds the compiled circuit zkeys bundled into the
mobile binary for the **Rust native prover** (`mobile/native-prover/`).
Each zkey is paired with a wasm-derived witness compiled into the
Cargo crate via `rust_witness::transpile_wasm` — see
`mobile/scripts/copy-zk-assets.sh`, which writes both this directory
and `mobile/native-prover/test-vectors/circom/`.

Files (regenerated by the circuit build, not committed to git — too
large at ~5–19 MB each):

- `deposit_final.zkey` — deposit proof
- `withdraw_final.zkey` — withdraw proof (full + partial)
- `claim_final.zkey` — claim proof
- `authorize_final.zkey` — authorize proof (market + limit orders)
- `cancel_final.zkey` — cancel proof

## Tier policy — mobile is TIER_16 only

The web stack (`apps/pay`, `apps/pro`, `frontend/`) ships authorize +
claim circuits in three tiers (16 / 64 / 128 recipients per
settlement) and routes per-run via `pickActiveTier(recipientCount)`.
**Mobile intentionally bundles only TIER_16 zkeys** for two reasons:

- **Bundle size.** The TIER_64 zkey is ~50 MB and TIER_128 is ~90 MB
  — bundling both would inflate the APK / IPA by ~140 MB before any
  product code, which is unacceptable for a mobile install footprint.
  Web frontends sidestep this with lazy-loaded static assets, but
  RN's asset pipeline doesn't translate cleanly.
- **Prove-time UX.** TIER_128 takes ~6–12 s on a mid-tier laptop;
  mobile-class hardware is meaningfully slower. TIER_16 (~1–2 s)
  already strains the in-app loading state; TIER_64 / TIER_128 push
  past a reasonable foreground-operation budget.

### Implications for callers

- A run with > 16 recipients must either be split into multiple
  tier-16 settlements (the current Pay multi-batch fallback path) or
  rejected at input time. Mobile UI should treat TIER_16 as the only
  available tier — don't call `pickActiveTier(...)` blindly.
- `mobile/scripts/copy-zk-assets.sh`'s `CIRCUITS` list is the
  source of truth for which circuits the mobile build bundles.
  `scripts/check-zk-artifacts.sh`'s `mobile_copies()` predicate must
  stay in lockstep — both list only the TIER_16 names.
- If a future tier ever ships for mobile (e.g. via on-demand asset
  download with a progress UI), update this README, the two scripts
  above, and the matching subsection in `docs/design/mobile/design.md`
  (currently § 10.1).

## Setup

```bash
# 1. Build circuits (requires the circom CLI).
cd circuits && bash scripts/build.sh

# 2. Mirror the freshly-built zkeys into this directory plus the
#    `mobile/native-prover/test-vectors/circom/` consumer of the
#    Rust witness transpilation.
cd ../mobile && bash scripts/copy-zk-assets.sh
```

Without these files, deposit / withdraw / claim / authorize / cancel
proof generation fails at runtime with an "asset not found" error
from `expo-asset`.
