# Native Prover — mopro binding for zkScatterDEX

Generates a React Native Turbo Module that runs Groth16 proving natively
instead of inside a `WKWebView`/snarkjs. Eliminates the iOS Simulator
dispatch-queue starvation that follows every WebView-side proof burst
(the stall that aborted `POST /api/authorize-orders` in #401/#404/#407).

Currently scoped to the `authorize` circuit (order submission). Other
circuits (`claim`, `cancel`, `deposit`) still run through the existing
WebView path until the spike is validated end-to-end.

## Regenerating the bindings

The xcframework static lib and the `test-vectors/circom/*.zkey|wasm`
build inputs are `.gitignore`d (too large + duplicated from
`circuits/build/`). After cloning, rebuild with:

```sh
# Prereqs (one-time)
cargo install mopro-cli
rustup target add aarch64-apple-ios-sim aarch64-apple-ios

# Copy circuit artifacts in (regenerated by `scripts/check-zk-artifacts.sh`)
cp ../../circuits/build/authorize_final.zkey test-vectors/circom/
cp ../../circuits/build/authorize_js/authorize.wasm test-vectors/circom/

# Build the iOS simulator slice (fastest; ~1-2 min on M-series)
mopro build \
  --platforms react-native \
  --mode release \
  --architectures aarch64-apple-ios-sim \
  --no-auto-update
```

For device / release builds, also include `aarch64-apple-ios`. Android
support is not wired yet.

## Layout

- `src/lib.rs` — registers circuits with `set_circom_circuits!`. Each
  entry needs a matching `rust_witness::witness!(name)` macro so mopro
  can pick up the witness calculator generated from the circom `.wasm`.
- `test-vectors/circom/` — `.zkey` + `.wasm` inputs, copied in from
  `circuits/build/`. Gitignored.
- `MoproReactNativeBindings/` — the generated RN package (JS + ObjC++ +
  C++ bindings + xcframework). Installed into `mobile/` as a local
  file dependency; see `mobile/package.json`.
