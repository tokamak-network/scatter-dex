zkScatter is a **privacy-preserving settlement protocol** for compliant
on-chain finance. The same primitives — a commitment pool, half-proof
authorization, a relayer network, identity gating — power **OTC
trading, private payments, and identity-gated distributions** on top of
one shared engine.

`@zkscatter/sdk` is the TypeScript toolkit for talking to every part of
that protocol — contracts, ZK proofs, relayer network, shared
orderbook, on-device note storage. **One package, one mental model**;
ship a wallet on web, mobile, or Node.

The same SDK powers the reference apps shipped by Tokamak —
[Pro](https://pro.zkscatter.xyz) (OTC),
[Pay](https://pay.zkscatter.xyz) (payments),
[Drop](https://drop.zkscatter.xyz) (drops), and Mobile. Anything those
apps do, your app can do, with the same primitives.

## How the flows compose

The same four steps cover every flow — what differs is *who* runs
which step. Deposit and claim are universal; authorize/match changes
shape per flow.

<Steps>
  <Step title="Deposit (universal)">
    The user signs a deposit ZK proof on-device. `CommitmentPool.deposit`
    inserts a Poseidon commitment hiding the owner secret, token, and
    amount. Same call for OTC, payments, drops.
  </Step>
  <Step title="Authorize / Issue">
    *OTC*: maker signs an EdDSA order + half-proof binding it to a leaf.
    *Payment*: sender signs a one-sided spend committing to a recipient
    address (regular or stealth — recipient's choice). *Drop*:
    distributor publishes a claims root committing to each recipient's
    address + amount.
  </Step>
  <Step title="Match / Submit">
    *OTC*: a relayer pairs two compatible half-proofs and submits
    `PrivateSettlement.settleAuth`. *Payment*: a relayer (or the
    sender) submits the spend directly. *Drop*: no matching — recipients
    claim independently.
  </Step>
  <Step title="Claim (universal)">
    The recipient's payout sits in `CommitmentPool` as a new commitment
    addressed to either a regular wallet or a stealth address (their
    earlier choice). They claim later — any time, gaslessly — by
    proving knowledge of the spending key. Up to 20 claims per
    transaction.
  </Step>
</Steps>
