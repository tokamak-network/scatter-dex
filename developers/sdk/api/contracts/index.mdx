# contracts

## Interfaces

| Interface | Description |
| ------ | ------ |
| [ClaimCallInputs](interfaces/ClaimCallInputs.mdx) | Public-input scalars the contract receives alongside the proof. These MUST match the values the caller passed to `generateClaimProof` — passing different scalars and the matching proof together would fail on-chain verification. |
| [BatchClaimItem](interfaces/BatchClaimItem.mdx) | - |
| [SettleAuthFees](interfaces/SettleAuthFees.mdx) | Per-side fee in token units the relayer charges, capped by the user's signed `maxFee` on each side. |
| [SettleAuthSide](interfaces/SettleAuthSide.mdx) | Per-side public material the contract re-checks against the proof. The `proof` carries hashes only; these scalars match the user-signed order. |

## Variables

| Variable | Description |
| ------ | ------ |
| [MAX\_CLAIM\_BATCH\_SIZE](variables/MAX_CLAIM_BATCH_SIZE.mdx) | Hard cap enforced by `PrivateSettlement.claimWithProofBatch`. Mirrors the contract's `MAX_CLAIM_BATCH_SIZE` constant; an oversized batch reverts on-chain after burning gas, so we catch it client-side. Callers chunk larger sets. |

## Functions

| Function | Description |
| ------ | ------ |
| [callCancel](functions/callCancel.mdx) | Send `PrivateSettlement.cancelPrivate(...)`. Anyone can submit on behalf of the user; the cancel proof binds `msg.sender` (the relayer) into the signed cancel message, so a different relayer cannot replay the proof. |
| [callClaimWithProof](functions/callClaimWithProof.mdx) | Send `PrivateSettlement.claimWithProof(...)`. Anyone can submit on behalf of the recipient (the address is encoded in the public signals), so a relayer can dispatch gaslessly. |
| [callClaimWithProofBatch](functions/callClaimWithProofBatch.mdx) | Batch variant. Reverts atomically if any element is invalid; caps at `MAX_CLAIM_BATCH_SIZE` (caller chunks larger sets). |
| [ensureAllowance](functions/ensureAllowance.mdx) | Approve `token` for `spender` to pull `amount` units. |
| [callDeposit](functions/callDeposit.mdx) | Send `CommitmentPool.deposit(...)`. Caller is responsible for the ERC-20 approval (see `ensureAllowance`). Returns the pending TransactionResponse so callers can show optimistic UI before `wait()`. |
| [callSettleAuth](functions/callSettleAuth.mdx) | Send `PrivateSettlement.settleAuth(...)`. Relayer-only — the contract enforces `msg.sender ∈ {maker.relayer, taker.relayer}`. |
