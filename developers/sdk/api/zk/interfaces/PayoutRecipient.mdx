# Interface: PayoutRecipient

Recipient line for a multi-recipient payout. The token is shared
 across the whole payout — see [SplitPayoutOpts.token](SplitPayoutOpts.mdx#token).

## Properties

| Property | Type | Description |
| ------ | ------ | ------ |
| <a id="recipient"></a> `recipient` | `string` | Ethereum address (0x-prefixed). |
| <a id="amount"></a> `amount` | `bigint` | Amount of `opts.token` to send to this recipient. |
| <a id="releasetime"></a> `releaseTime` | `bigint` | Earliest unix-second the recipient can claim. |
| <a id="secret"></a> `secret?` | `bigint` | Optional pre-computed per-claim secret. When omitted, [splitPayout](../functions/splitPayout.mdx) draws one from [SplitPayoutOpts.generateSecret](SplitPayoutOpts.mdx#generatesecret) (defaulting to [randomFieldElement](../functions/randomFieldElement.mdx)). |
