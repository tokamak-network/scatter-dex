# Interface: StoredNote

Persistent note record. Carries everything an app needs to spend
 (`note` preimage), display (`symbol`, `amount`, `label`), and
 reconcile against chain state (`leafIndex`, `txHash`, `chainId`).

 BigInts are kept native here; adapters handle wire-format
 serialization (hex strings) at the storage boundary.

## Properties

| Property | Type | Description |
| ------ | ------ | ------ |
| <a id="id"></a> `id` | `string` | Stable per-record id (uuid). |
| <a id="label"></a> `label` | `string` | Display label, e.g. `lot-1`. |
| <a id="symbol"></a> `symbol` | `string` | Token symbol shown in the UI. |
| <a id="amount"></a> `amount` | `string` | Display amount string (already formatted for the UI). Not used for math — `note.amount` is the canonical raw value. |
| <a id="note"></a> `note` | [`CommitmentNote`](../../zk/interfaces/CommitmentNote.mdx) | Full commitment-note preimage. The secret material that lets the holder spend the deposit. |
| <a id="commitment"></a> `commitment` | `bigint` | Poseidon commitment derived from `note`. Cached so callers don't recompute on every render. |
| <a id="leafindex"></a> `leafIndex` | `number` | On-chain leaf index. `-1` when the deposit's `CommitmentInserted` event hasn't been reconciled yet. |
| <a id="txhash"></a> `txHash?` | `string` | Deposit transaction hash, when known. |
| <a id="chainid"></a> `chainId?` | `number` | Chain id this note belongs to — apps should only show notes for the active network. |
| <a id="createdat"></a> `createdAt` | `number` | When the note was added (ms epoch). |
