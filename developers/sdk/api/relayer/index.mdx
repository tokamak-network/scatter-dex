# relayer

## Classes

| Class | Description |
| ------ | ------ |
| [RelayerClient](classes/RelayerClient.mdx) | Thin HTTP client for one relayer's API. No retry, no caching; callers compose those concerns. Errors throw with the relayer's `error` body field when present, falling back to the HTTP status line. Every method accepts an optional `signal` so callers can cancel from a UI's `AbortController`. |

## Type Aliases

| Type Alias | Description |
| ------ | ------ |
| [OperatorStatus](type-aliases/OperatorStatus.mdx) | UI-friendly status derived from the on-chain `relayers()` row. - `unregistered`: never registered on this registry - `active`: registered, not in the exit cool-down - `cooldown`: requested exit, waiting out the 7-day cool-down - `offline`: registered + exit executed (the row's `active` flag is false but `registeredAt` is non-zero) |

## Variables

| Variable | Description |
| ------ | ------ |
| [EXIT\_COOLDOWN\_SECONDS](variables/EXIT_COOLDOWN_SECONDS.mdx) | Mirrors `RelayerRegistry.EXIT_COOLDOWN`. The contract exposes it as a public constant; we mirror the value here so callers can render the cool-down countdown without an extra RPC read. Keep in sync if governance ever rewrites the constant. |
| [MAX\_RELAYER\_FEE\_BPS](variables/MAX_RELAYER_FEE_BPS.mdx) | Maximum per-trade fee a relayer may register, in basis points. Mirrors `RelayerRegistry.MAX_FEE` (50 bps) — kept in sync here so the SDK can reject invalid input before a wallet prompt and so consumer apps don't redeclare the magic number. |

## Functions

| Function | Description |
| ------ | ------ |
| [callExceptionErrorName](functions/callExceptionErrorName.mdx) | Read the named error off ethers v6 contract-call exceptions when the ABI carries the matching error fragment. Falls back to null so callers can substring-match the message instead. Shared by every per-contract `explainXxxError` in the relayer module. |
| [unwrapEthersError](functions/unwrapEthersError.mdx) | Best-effort unwrap of an ethers v6 error to a human-readable string. v6 surfaces the user-facing summary on `shortMessage`, the decoded revert reason on `reason`, and the underlying provider error on `info.error.message`. Fall through these in priority order before landing on the raw `Error.message` so callers (read-side error banners, write-side `explainXxxError`) always have the most descriptive string available. |
| [loadFeeVaultBalances](functions/loadFeeVaultBalances.mdx) | Read every passed token's claimable balance for `operator` in one shot. Caller decides which tokens to query — typically the network's `tokens` whitelist — so this stays a pure batched read with no contract-side enumeration assumed. Tokens whose address is the zero sentinel (unconfigured slot in a partially deployed network config) are skipped. |
| [claimRelayerFees](functions/claimRelayerFees.mdx) | Submit `claim(token)` to pull the operator's accrued balance for a single token. Reverts with `NothingToClaim` when the balance is zero — gate the button on a non-zero balance read so the wallet prompt never lands on a guaranteed-fail tx. |
| [explainFeeVaultError](functions/explainFeeVaultError.mdx) | Map FeeVault custom-error reverts to user-facing copy. Falls back to the unwrapped ethers v6 message so unexpected errors still surface a useful string instead of `[object Object]`. |
| [loadIdentityVerification](functions/loadIdentityVerification.mdx) | Read an account's verification status from an arbitrary IdentityGate contract. zkScatter's Dual-CA architecture deploys one gate per CA (User CA = privacy-preserving, Relayer CA = full-disclosure), so callers pass the gate address explicitly rather than letting the SDK guess one. Pure read. |
| [updateRelayerInfo](functions/updateRelayerInfo.mdx) | Submit `updateInfo(url, fee)` — operator-self-service edit of endpoint URL + per-trade fee. Validates the fee range up front for the same UX reason `registerRelayer` does. |
| [addRelayerBond](functions/addRelayerBond.mdx) | Submit `addBond()` with the given top-up as `msg.value`. Rejects zero / negative amounts before a wallet prompt. |
| [requestRelayerExit](functions/requestRelayerExit.mdx) | Submit `requestExit()` — flips the operator into the cool-down window. New orders stop routing immediately; bond becomes withdrawable via `executeRelayerExit` after `EXIT_COOLDOWN_SECONDS`. |
| [executeRelayerExit](functions/executeRelayerExit.mdx) | Submit `executeExit()` — finalises the exit and returns the bond. Will revert with `CooldownNotPassed` until the cool-down window has elapsed; gate the button on `cooldownReadyAt`. |
| [loadOperatorRow](functions/loadOperatorRow.mdx) | Read the on-chain registry row for `account`. Returns the full set of operator-scoped state every dashboard / profile / treasury page needs, plus a derived `status` for UI gating. Pure read. |
| [sanitizeProfile](functions/sanitizeProfile.mdx) | Sanitize the `profile` block returned by an arbitrary relayer's `/api/info`. We trust nothing here: keep only known string fields, enforce a length cap, reject URL fields whose scheme isn't on the allowlist. Guards against: - UI crashes from non-string fields breaking `.replace` etc. - rendered-link XSS via `javascript:` / `data:` schemes - DOS via huge strings |
| [loadRegistrationStatus](functions/loadRegistrationStatus.mdx) | Read the prerequisite state for relayer registration: identity verification, prior active-relayer status, and the registry's current minimum bond. Pure read. |
| [registerRelayer](functions/registerRelayer.mdx) | Submit `register(url, fee)` with the given bond as `msg.value`. Validates the fee range and bond format up front so the user sees a clean error before a wallet prompt. Returns the transaction response; caller awaits `.wait()`. |
| [explainRegistryError](functions/explainRegistryError.mdx) | Map known registry custom errors (decoded by ethers from the ABI fragments on `RELAYER_REGISTRY_ABI`) and local validation errors thrown from any registry write helper to user-friendly copy. Falls back to the raw message when no rule matches so callers don't swallow unexpected errors. `minBond` is only used when the error is `InsufficientBond`; pass `0n` if unknown. |
| [loadActiveRelayers](functions/loadActiveRelayers.mdx) | Read the registry contract's active list and return the full on-chain row for each. Pure read; no side effects. |
| [loadRelayersWithApiInfo](functions/loadRelayersWithApiInfo.mdx) | Combine on-chain registry data with a live `/api/info` probe per relayer. Probes run in parallel; offline relayers come back with `online: false` and `api: undefined`. |
