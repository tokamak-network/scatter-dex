# Interface: WalletState

## Properties

| Property | Type | Description |
| ------ | ------ | ------ |
| <a id="account"></a> `account` | `string` \| `null` | Connected EOA, lowercased. Null when disconnected. |
| <a id="chainid"></a> `chainId` | `number` \| `null` | Chain id reported by the wallet. May differ from `network.chainId` while the user is on the wrong chain. |
| <a id="signer"></a> `signer` | `any` | Wallet-backed signer for transactions. Null when disconnected. |
| <a id="provider"></a> `provider` | `any` | Browser provider wrapping the wallet's EIP-1193. |
| <a id="readprovider"></a> `readProvider` | `JsonRpcProvider` | Read-only RPC provider built from `network.rpcUrl`. Always available regardless of wallet state. |
| <a id="walletname"></a> `walletName` | `string` \| `null` | Best-effort wallet vendor name; null when disconnected. |
| <a id="connecterror"></a> `connectError` | `string` \| `null` | Last error from a `connect()` attempt — covers both the "no wallet detected" case and user rejections. |
| <a id="connect"></a> `connect` | () => `Promise`\<`void`\> | Trigger the wallet's account-request flow. |
| <a id="disconnect"></a> `disconnect` | () => `void` | Drop the connected account from app state. (Most wallets don't expose a programmatic disconnect; this clears local state only.) |
